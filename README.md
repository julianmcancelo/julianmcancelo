### `// 02` — PROYECTOS DESTACADOS

</div>

<br>

<div align="center">

```
┌─────────────────────────────────────────────────────────────────────────┐
│  >_ STOCKAR                                              [ACTIVO]  ████  │
│  Sistema integral de gestión de inventario y control operativo          │
│  Multiplataforma · Tiempo real · Offline-first                          │
└─────────────────────────────────────────────────────────────────────────┘
```

</div>

**El problema que resuelve:** operaciones con inventario disperso, sin visibilidad en tiempo real y con riesgo de desincronización entre canales. STOCKAR centraliza y sincroniza todo desde una sola fuente de verdad.

**Arquitectura:**

```bash
# stockar/config ──────────────────────────────────────────────────────────
LAYERS = {
  "mobile":     "React Native + Expo   →  operadores en campo",
  "web":        "React + Vite          →  dashboard administrativo",
  "backend":    "Google Apps Script    →  lógica de negocio + integraciones",
  "data":       "Google Sheets + API   →  persistencia + sincronización"
}

FEATURES = [
  "Sincronización en tiempo real entre capas",
  "Modo offline con reconciliación automática",
  "Control de stock, movimientos y trazabilidad",
  "Reportes operativos y alertas configurables",
  "Acceso por roles con permisos diferenciados"
]

ENFOQUE = "Offline-first · Alta disponibilidad · Operación continua"
# ─────────────────────────────────────────────────────────────────────────
```

**Desafíos técnicos resueltos:** gestión de conflictos en sincronización offline, diseño de una API liviana sobre Google Apps Script que soporte múltiples clientes simultáneos, y UX optimizada para operadores en campo con conectividad inestable.

<br>

<div align="center">

```
┌─────────────────────────────────────────────────────────────────────────┐
│  >_ BITÁCORA DOCENTE                                     [ACTIVO]  ████  │
│  Digitalización del registro y seguimiento del trabajo docente          │
│  Usabilidad · Trazabilidad · Gestión centralizada                       │
└─────────────────────────────────────────────────────────────────────────┘
```

</div>

**El problema que resuelve:** el trabajo docente está fragmentado en papeles, planillas sueltas y memoria. Sin registro estructurado, la continuidad y la calidad de la información se pierden. Bitácora Docente cambia eso.

**Arquitectura:**

```bash
# bitacora-docente/config ─────────────────────────────────────────────────
OBJETIVO = "Centralizar el registro, planificación y seguimiento docente"

FEATURES = [
  "Registro cronológico de clases y contenidos",
  "Seguimiento de alumnos y asistencia",
  "Planificación y gestión de evaluaciones",
  "Historial exportable por período o materia",
  "Interfaz diseñada para bajo roce operativo"
]

VALOR = {
  "para_docentes":    "Menos carga administrativa, más foco pedagógico",
  "para_gestión":     "Información confiable y accesible en tiempo real",
  "para_continuidad": "Registro estructurado que no depende de una persona"
}
# ─────────────────────────────────────────────────────────────────────────
```

**Enfoque de diseño:** UX pensada para usuarios sin perfil técnico. Flujo de uso mínimo, información siempre accesible y exportable. La herramienta se adapta al docente, no al revés.
